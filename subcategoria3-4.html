¿QUÉ ES?

Un formulario HTML es una sección de una página web que permite a los usuarios interactuar con el sitio y enviar información al servidor.

Elemento: Form

Es el elemento principal del formulario y lo define como tal. Debe tener los atributos action y method especificados.
action: Indica la URL del script del lado del servidor que procesará los datos del formulario.
method: Define el método HTTP que se utilizará para enviar los datos (generalmente GET o POST). Puede contener otros elementos HTML dentro de él, como etiquetas de entrada, botones y campos de texto.

Etiquetas de Entrada: Input

Permiten a los usuarios ingresar datos en el formulario. Existen diferentes tipos de etiquetas de entrada, como:
text: Para texto general.
email: Para direcciones de correo electrónico.
password: Para contraseñas (se oculta el texto introducido).
checkbox: Para casillas de verificación.
radio: Para botones de opción.
number: Para números.
date: Para fechas.
file: Para subir archivos.
Cada etiqueta de entrada debe tener atributos como type, name (para identificar el dato) y value (valor predeterminado).

Botones de Envio: Submit

Permiten al usuario enviar los datos del formulario al servidor. Se pueden personalizar con texto como "Enviar", "Registrarse", etc. Cuando se hace clic en el botón, se desencadena el evento submit del formulario.

Etiquetas de Texto: Label

Sirven para asociar una etiqueta descriptiva con un elemento de entrada. Se utilizan junto al atributo for de la etiqueta de entrada para vincularlas. Mejoran la accesibilidad y la usabilidad del formulario.

Campos de Texto: TextArea

Permiten al usuario ingresar texto de varias líneas. Se utilizan para comentarios, descripciones detalladas, etc. Tienen atributos como rows (número de filas visibles) y cols (número de columnas).

Elementos de Seccion: Select

Presentan al usuario una lista de opciones para seleccionar. Se utilizan para categorías, preferencias, etc. Contienen etiquetas 'option' dentro de ellas para definir las opciones disponibles.

Atributos Adicionales

required: Indica que el campo es obligatorio y debe completarse.
disabled: Inhabilita el campo, impidiendo que el usuario lo modifique.
readonly: Permite que el usuario vea el valor del campo, pero no lo modifique.
placeholder: Muestra un texto de sugerencia dentro del campo cuando está vacío.
pattern: Define una expresión regular para validar el formato del dato ingresado.

Código de Ejemplo:

                
    <form action="procesar_registro.php" method="post" id="registroForm">
        <div class="form-group">
            <label for="nombre">Nombre Completo:</label>
            <input type="text" id="nombre" name="nombre" class="form-control" required>
         </div>

         <div class="form-group">
             <label for="correo">Correo electrónico: </label>
             <input type="email" id="correo" name="correo" class="form-control" required>
         </div>

         <div class="form-group">
             <label for="contrasena">Contraseña: </label>
             <input type="password" id="contrasena" name="contrasena" class="form-control" required>
         </div>

         <div class="form-group">
             <label for="confirmarContrasena">Confirmar contraseña: </label>
             <input type="password" id="confirmarContrasena" name="confirmarContrasena" class="form-control" required>
         </div>

         <div class="form-group">
             <label for="pais">País: </label>
             <select id="pais" name="pais" class="form-control">
                 <option value="mx">México </option>
                 <option value="es">España </option>
                 <option value="us">Estados Unidos </option>
             </select>
        </div>

        <div class="form-group">
            <label for="aceptoTerminos">Acepto los términos y condiciones</label>
            <input type="checkbox" id="aceptoTerminos" name="aceptoTerminos" class="form-control" required>
        </div>

        <button type="submit" class="btn btn-primary">Registrarse</button>
    </form>
